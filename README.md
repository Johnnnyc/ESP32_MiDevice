# ESP32米家智能设备

## 项目概述

本项目将ESP32开发板打造成一个完全兼容米家生态系统的智能BLE设备。设备支持4路IO开关控制和DHT11温湿度传感器数据采集，可以直接在米家APP中添加和控制。

## 功能特性

- **BLE设备**：ESP32作为BLE设备广播，可被青萍盲网关发现
- **米家系统兼容**：支持米家系统的设备识别和控制协议
- **真实传感器数据**：使用DHT11采集真实温湿度数据
- **多开关控制**：支持多个IO口开关的独立控制
- **实时数据上报**：向网关实时上报温湿度数据和开关状态
- **远程控制**：接收并响应米家系统的控制命令
- **设备状态监控**：持续监控并上报设备状态
- **异常检测**：传感器读取失败时的错误处理

## 硬件要求

- ESP32开发板（ESP32-WROOM-32或ESP32-DevKitC推荐）
- DHT11温湿度传感器
- 杜邦线若干
- 外部电源或USB数据线（用于供电和调试）
- 可选：继电器模块或LED灯（用于开关控制演示）

## 软件要求

- Arduino IDE（1.8.13或更高版本）
- ESP32 Arduino核心支持包
- BLE库（Arduino ESP32内置）
- DHT传感器库（Adafruit DHT Sensor Library）

## 安装与配置

### 1. 安装Arduino IDE和ESP32支持包

1. 下载并安装Arduino IDE：[https://www.arduino.cc/en/software](https://www.arduino.cc/en/software)
2. 打开Arduino IDE，进入**文件** > **首选项**
3. 在"附加开发板管理器网址"中添加：`https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json`
4. 进入**工具** > **开发板** > **开发板管理器**
5. 搜索"esp32"并安装最新版本

### 2. 安装DHT传感器库

1. 打开Arduino IDE
2. 进入**工具** > **管理库**
3. 搜索"DHT sensor library"并安装Adafruit DHT Sensor Library
4. 同样安装"Adafruit Unified Sensor"库（DHT库的依赖）

### 3. 硬件连接

#### DHT11传感器连接
| DHT11引脚 | ESP32引脚 |
|-----------|-----------|
| VCC       | 3.3V      |
| GND       | GND       |
| DATA      | GPIO4     |

#### 开关IO口连接
| 开关编号 | ESP32引脚 | 用途说明 |
|----------|-----------|----------|
| 开关1    | GPIO5     | 控制设备1 |
| 开关2    | GPIO18    | 控制设备2 |
| 开关3    | GPIO19    | 控制设备3 |
| 开关4    | GPIO21    | 控制设备4 |

**注意**：如果需要控制高电压设备（如灯泡、电机等），请使用继电器模块隔离，不要直接将高电压连接到ESP32的IO口。

### 4. 配置项目

1. 打开`ESP32_MiDevice.ino`文件
2. 根据需要修改以下配置参数：

```cpp
// 设备配置
#define DEVICE_NAME "ESP32_MiDevice"  // 设备名称
#define DEVICE_MODEL "esp32.sensor_switch.v1"  // 设备型号（可自定义）

// DHT11传感器配置
#define DHT_PIN 4       // DHT11连接到GPIO4
#define DHT_TYPE DHT11  // DHT传感器类型

// IO口开关配置
#define SWITCH_COUNT 4  // 支持的开关数量
#define SWITCH_PINS {5, 18, 19, 21}  // 开关连接的GPIO口
```

3. 选择正确的开发板型号和端口
4. 点击"上传"按钮将代码烧录到ESP32

### 5. 米家APP添加设备

1. **准备工作**：
   - 确保青萍盲网关或其他米家BLE网关已连接到米家APP
   - 确保手机蓝牙已开启
   - 确保ESP32设备已通电并正常运行

2. **添加设备步骤**：
   - 打开米家APP，点击右上角「+」按钮
   - 选择「添加设备」
   - 选择「自动发现设备」或点击「手动添加」>「传感器」>「温湿度传感器」
   - 等待APP发现设备，点击「连接」
   - 给设备命名（如「ESP32智能开关」），选择设备位置
   - 完成添加，设备将显示在米家APP首页

3. **设备控制**：
   - 在米家APP中点击设备图标进入控制界面
   - 可以看到4个开关按钮，点击即可控制对应的IO口
   - 界面将显示当前温湿度数据
   - 可以设置自动化场景（如温度高于25°C时打开风扇）

## 工作原理

### 1. BLE广播

ESP32作为BLE设备广播，使用米家设备的标准UUID：
- 服务UUID：`0000fe95-0000-1000-8000-00805f9b34fb`
- 特征UUID：`00000001-0000-1000-8000-00805f9b34fb`

这些UUID是米家设备的标准UUID，确保设备能被青萍盲网关或其他米家BLE网关识别。

### 2. 设备发现与认证

- 青萍盲网关或其他米家BLE网关扫描BLE设备时，会发现ESP32设备
- 设备响应米家网关的认证请求
- 网关读取设备信息（名称、型号、MAC地址等）
- 网关将设备信息上报到米家系统
- 用户可以在米家App中看到并控制该设备

### 3. 数据交互

- **设备到网关**：定期采集并上报温湿度数据、开关状态、电量等信息
- **网关到设备**：接收米家APP的控制命令，如开关控制
- **数据显示**：上报的数据在米家APP中实时显示

## 米家协议支持与控制

### 协议特性

设备完全支持米家BLE设备协议，包括：
- **标准服务UUID**：`0000fe95-0000-1000-8000-00805f9b34fb`
- **标准特征UUID**：`00000001-0000-1000-8000-00805f9b34fb`
- **设备认证**：响应米家网关的认证请求
- **数据加密**：预留加密密钥存储（deviceKey）

### 在米家APP中控制

1. **开关控制**：
   - 点击APP中的开关按钮即可控制对应的IO口
   - 支持单路独立控制

2. **传感器数据查看**：
   - APP界面实时显示当前温湿度数据
   - 支持查看历史数据趋势

3. **自动化场景**：
   - 温度控制：如温度高于25°C时打开风扇
   - 湿度控制：如湿度高于60%时打开除湿机
   - 定时控制：如定时开关灯光或电器

### 支持的底层控制命令

#### 单个开关控制
- **命令**：`0x01`
- **格式**：`0x01 + switchIndex + state`
- **参数说明**：
  - `switchIndex`：开关索引（0-3，对应开关1-4）
  - `state`：`0x01`（开启）/ `0x00`（关闭）

#### 读取传感器数据
- **命令**：`0x02`
- **说明**：立即读取并上报传感器数据

#### 批量开关控制
- **命令**：`0x03`
- **格式**：`0x03 + state1 + state2 + state3 + state4`
- **参数说明**：
  - `state1-state4`：每个开关的状态（`0x01`/`0x00`）

## 测试方法

1. **硬件测试**：
   - 检查DHT11传感器连接是否正确
   - 检查4路IO开关连接是否正确

2. **软件测试**：
   - 上传代码到ESP32开发板
   - 打开串口监视器（波特率115200）
   - 观察设备是否成功初始化BLE并开始广播
   - 观察设备是否定期采集和上报传感器数据
   - 观察设备是否响应米家APP的连接请求

3. **米家APP功能测试**：
   - 在米家APP中添加设备
   - 点击APP中的开关按钮，检查对应的IO口是否有正确的开关动作
   - 检查米家APP中显示的温湿度数据是否与实际环境一致
   - 设置自动化场景，检查是否能按预期执行

4. **BLE调试**：
   - 使用BLE调试工具（如nRF Connect）扫描设备
   - 可以看到设备名称为「ESP32_MiDevice」
   - 可以查看服务UUID和特征UUID是否符合米家标准
   - 可以读取设备信息和状态数据

## 传感器数据说明

- **温度范围**：0-50℃（DHT11的测量范围）
- **湿度范围**：20-90%RH（DHT11的测量范围）
- **更新频率**：默认每5秒更新一次传感器数据
- **精度**：温度±2℃，湿度±5%RH（DHT11的测量精度）

## 扩展与定制

### 修改开关数量和引脚

1. 修改`SWITCH_COUNT`定义以更改支持的开关数量
2. 修改`SWITCH_PINS`数组以更改开关连接的GPIO口

### 调整传感器更新频率

1. 在`loop()`函数中找到以下代码：
```cpp
if (millis() - lastSensorUpdate > 5000) {  // 每5秒更新一次传感器数据
```
2. 修改`5000`（毫秒）为您需要的更新间隔

### 添加新的传感器

1. 安装相应的传感器库
2. 连接传感器到ESP32的GPIO口
3. 在代码中初始化传感器
4. 在`readSensorData()`函数中添加传感器数据读取逻辑
5. 在`updateDeviceData()`函数中添加数据上报逻辑

### 支持新的控制命令

1. 在`parseControlCommand()`函数中添加新的命令解析逻辑
2. 实现对应的设备控制功能

## 注意事项

1. 确保ESP32与青萍盲网关在蓝牙有效范围内（通常10-20米）
2. 该项目使用模拟传感器数据，实际应用中可以连接真实传感器
3. 设备型号可以自定义，但建议使用与米家系统兼容的格式
4. 如果设备无法被发现，可以尝试重启ESP32或青萍盲网关

## 故障排除

### 设备无法在米家APP中被发现
- 检查ESP32是否正常通电
- 检查手机蓝牙是否开启
- 检查青萍盲网关或其他米家BLE网关是否在线
- 检查设备广播是否正常（查看串口监视器输出）
- 确保设备与网关在有效距离内（通常10-20米）

### 传感器数据异常
- 检查DHT11传感器连接是否正确
- 检查DHT11传感器是否损坏
- 增加传感器读取重试次数

### 开关控制无效
- 检查IO口连接是否正确
- 检查继电器（如果使用）是否正常工作
- 检查设备是否与米家网关保持连接

### 设备频繁离线
- 检查ESP32电源是否稳定
- 检查设备与网关之间的距离是否过远
- 减少环境中的蓝牙干扰

### 无法添加到米家APP
- 确保使用的是最新版本的米家APP
- 尝试重启青萍盲网关或其他米家BLE网关
- 尝试重置ESP32设备（重新上传代码）
